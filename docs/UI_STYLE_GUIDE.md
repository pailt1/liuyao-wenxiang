# UI_STYLE_GUIDE.md

## 目标

根据 docs/ui-reference 中的 5 张效果图，优化“六爻问象”微信小程序 UI。

目标是先达到效果图 70%–80% 的视觉还原，不要求完全复刻复杂插画，但整体风格、布局、配色、卡片层级、六爻线条展示要接近。

## 总体风格

关键词：

- 现代东方感
- 米白纸张背景
- 黑色文字
- 暗金色点缀
- 圆角卡片
- 留白充分
- 克制、安静、高级
- 不要江湖算命风
- 不要大红大紫
- 不要过度玄学

## 页面要求

### 首页 index

参考：docs/ui-reference/01-home.png

需要包含：

- 顶部标题：六爻问象
- 视觉主标题：六爻问象
- 副标题：电子摇卦，静心观象
- 问题输入卡片
- 问题类型选择
- 提示条：一事一问，静心起卦；程序负责排盘，AI辅助解读
- 主按钮：开始摇卦
- 底部免责声明

首页重点：

- 输入区域要大
- 按钮要明显
- 顶部要有东方感视觉，但可以用 CSS 简化
- 不需要复杂插画，先用淡色背景纹理、云纹、山形渐变替代

### 摇卦页 shake

参考：docs/ui-reference/02-shake.png

需要包含：

- 顶部标题：摇卦中
- 问题卡片
- 当前进度：第 X / 6 爻
- 三枚铜钱区域
- 六爻生成列表
- 当前正在生成的爻高亮
- 已生成的爻显示完成状态
- 按钮：继续摇卦、重新开始
- 底部提示：六爻从下往上生成，初爻在最下，上爻在最上

摇卦页重点：

- 铜钱可以先用 CSS 圆形实现
- 动画可以先做简单旋转或上下浮动
- 六爻线条必须用代码绘制，不要用图片

### 结果页 result

参考：docs/ui-reference/03-result.png

需要包含：

- 顶部标题：解卦结果
- 问题摘要卡片
- 六爻图形卡片
- 本卦、变卦、动爻信息
- AI 结论卡片
- 分析卡片：
  - 本卦含义
  - 变卦趋势
  - 动爻提示
  - 现实建议
- 按钮：
  - 保存记录
  - 分享结果
  - 重新起卦
- 底部免责声明

结果页重点：

- 六爻图形要清晰
- 动爻用红色或暗红色标识
- 本卦、变卦、动爻信息要比正文更突出
- AI 解读不要堆成长文章，做成卡片分段

### 历史记录页 history

参考：docs/ui-reference/04-history.png

需要包含：

- 顶部标题：历史记录
- 分类筛选：全部、考试、感情、事业
- 历史记录卡片列表
- 每张卡片显示：
  - 问题
  - 时间
  - 类型
  - 本卦 → 变卦
  - 一句话摘要
  - 查看详情
- 底部或悬浮按钮：重新起卦

历史页重点：

- 卡片信息要清晰
- 不要太挤
- 每条记录看起来像可点击列表项

### 记录详情页 detail

参考：docs/ui-reference/05-detail.png

需要包含：

- 顶部标题：记录详情
- 问题信息卡片
- 本卦 / 变卦 / 动爻卡片
- AI 解读分段
- 底部操作：
  - 再次起卦
  - 复制文本
  - 删除记录
- 免责声明

详情页重点：

- 和结果页风格保持一致
- 更偏“复盘阅读”
- 按钮层级要清楚，删除按钮使用红色弱强调

## 组件规范

### 卡片

- 圆角：20rpx–32rpx
- 背景：#FFFDF8
- 边框：1rpx solid #E8DDCB
- 阴影：轻微，不要重阴影
- 内边距：28rpx–36rpx

### 按钮

主按钮：

- 背景：暗金色渐变或纯色 #B8872B
- 文字：白色
- 圆角：24rpx–32rpx
- 高度：88rpx–104rpx

次按钮：

- 背景：透明或 #FFFDF8
- 边框：#B8872B
- 文字：#A9781F

### 六爻线条

必须用 View + CSS 绘制。

阳爻：
一条完整横线。

阴爻：
左右两段横线，中间断开。

动爻：
在该爻右侧显示小圆点或红色标记。

不要用图片绘制六爻。

## 禁止事项

- 不要修改六爻计算逻辑
- 不要修改本卦、变卦、动爻算法
- 不要改 AI prompt 逻辑
- 不要把 UI 图片当成页面背景整张贴上
- 不要使用远程图片
- 不要引入复杂第三方 UI 框架
- 不要在前端写 API Key
