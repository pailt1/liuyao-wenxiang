# AGENTS.md

## 项目名称

六爻问象 / 象问

## 项目定位

本项目是一个结合传统六爻文化与 AI 解读的微信小程序。

产品定位是：
- 传统文化学习工具
- 易经象数辅助解读工具
- 决策反思辅助工具

不要写成：
- 算命大师
- 绝对预测命运
- 保证结果
- 改命转运
- 恐吓用户的玄学工具

所有 AI 解读必须带免责声明。

---

## 技术栈

- 微信原生小程序
- WXML / WXSS / JavaScript
- 微信云开发
- 云函数 Node.js
- 云数据库
- 后续接入 OpenAI / DeepSeek / 通义等大模型 API

---

## 最重要的规则

1. 六爻必须从下往上排列。
2. 第 1 爻是初爻，在最下面。
3. 第 6 爻是上爻，在最上面。
4. 程序负责排盘，AI 只负责解释。
5. 不允许让 AI 自己识别卦象。
6. API Key 不能放在小程序前端。
7. AI 解读不能说“必然成功”“一定失败”“百分百准确”等绝对化判断。
8. 每次解读必须提示“仅供传统文化学习与自我反思参考”。

---

## 代码结构原则

核心排盘逻辑必须放在：

/utils/hexagram.js

64卦数据放在：

/data/hexagrams.json

AI 提示词放在：

/docs/AI_PROMPT.md

六爻规则放在：

/docs/HEXAGRAM_RULES.md

不要把核心排盘逻辑散落在页面 JS 文件里。

---

## 修改代码前请先做

1. 先阅读 AGENTS.md。
2. 如果涉及排盘，先阅读 docs/HEXAGRAM_RULES.md。
3. 如果涉及 AI 解卦，先阅读 docs/AI_PROMPT.md。
4. 修改前说明准备改哪些文件。
5. 修改后列出变更点。
6. 涉及排盘逻辑时，必须给出测试案例。

---

## 测试重点

重点测试：

1. 六爻顺序是否从下往上。
2. 本卦是否计算正确。
3. 变卦是否计算正确。
4. 动爻是否识别正确。
5. 无动爻时，本卦和变卦是否一致。
6. 多动爻时，所有动爻是否正确变化。
7. AI 是否只根据程序计算结果解释，而不是自己重新算卦。
